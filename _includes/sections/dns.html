{% t Terms %}

  • {% t DNS-over-TLS (DoT) - A security protocol for encrypted DNS on a dedicated port 853. Some providers support port 443 which generally works everywhere while port 853 is often blocked by restrictive firewalls. DoT has two modes: %}
    • {% t Oppurtunistic mode: the client attempts to form a DNS-over-TLS connection to the server on port 853 without performing certificate validation. If it fails, it will use unencrypted DNS.%}
    • {% t Strict mode: the client connects to a specific hostname and performs certificate validation for it. If it fails, no DNS queries are made until it succeeds. %}
  • {% t DNS-over-HTTPS (DoH) - Similar to DoT, but uses HTTPS instead, being indistinguishable from "normal" HTTPS traffic on port 443.%}
  • {% t DNSCrypt - An older yet robust method of encrypting DNS. %}

{% t How to verify DNS is encrypted %}

  • {% t DoH / DoT %}
    • {% t Check DNSLeakTest.com.%}
    • {% t Check the website of your DNS provider. They may have a page for telling "you are using our DNS." Examples include AdGuard and Cloudflare.%}
    • {% t If using Firefox's trusted recursive resolver (TRR), navigate to about:networking#dns. If the TRR column says "true" for some fields, you are using DoH.%}
  • {% t dnscrypt-proxy - Check dnscrypt-proxy's wiki on how to verify that your DNS is encrypted.%}
  • {% t DNSSEC - Check DNSSEC Resolver Test by Matth√§us Wander.%}
  • {% t QNAME Minimization - Run dig +short txt qnamemintest.internet.nl from the command-line (taken from this NLnet Labs presentation). If you are on Windows 10, run Resolve-DnsName -Type TXT -Name qnamemintest.internet.nl from the PowerShell. You should see this display: "HOORAY - QNAME minimisation is enabled on your resolver :)!" %}

{% t Software Suggestions and Additional Information %}

  • {% t Encrypted DNS clients for desktop: %}
    • {% t Firefox comes with built-in DoH support with Cloudflare set as the default resolver, but can be configured to use any DoH resolver.%} {% t Currently Mozilla is conducting studies before enabling DoH by default for all US-based Firefox users. %}
      • {% t DNS over HTTPS can be enabled in Menu -> Preferences (about:preferences) -> Network Settings -> Enable DNS over HTTPS. Set "Use Provider" to "Custom", and enter your DoH provider's address. %}
      • {% t Advanced users may enable it in about:config by setting network.trr.custom_uri and network.trr.uri as the address you find from the documentation of your DoH provider and network.trr.mode as 2. It may also be desirable to set network.security.esni.enabled to True in order to enable encrypted SNI and make sites supporting ESNI a bit more difficult to track. %}
  • {% t Encrypted DNS clients for mobile: %}
    • {% t Android 9 comes with a DoT client by default.%}
      • {% t We recommend selecting Private DNS provider hostname and entering the DoT address from documentation of your DoT provider to enable strict mode (see Terms above).%}
    • {% t DNSCloak %} - {% t An open-source DNSCrypt and DoH client for iOS by the Center for the Cultivation of Technology gemeinnuetzige GmbH.%}
    • {% t Nebulo %} - {% t An open-source application for Android supporting DoH and DoT. It also supports caching DNS responses and locally logging DNS queries.%}
  • {% t Local DNS servers: %}
    • {% t Stubby %} - {% t An open-source application for Linux, macOS, and Windows that acts as a local DNS Privacy stub resolver using DoT.%}
    • {% t Unbound %} - {% t a validating, recursive, caching DNS resolver. It can also be ran network-wide and has supported DNS-over-TLS since version 1.7.3.%}
      • {% t See also Actually secure DNS over TLS in Unbound on ctrl.blog.%}
  • {% t Network-wide DNS servers:%}
    • {% t Pi-hole %} - {% t A network-wide DNS server mainly for the Raspberry Pi. Blocks ads, tracking, and malicious domains for all devices on your network. %}
    • {% t NoTrack %} - {% t A network-wide DNS server like Pi-hole for blocking ads, tracking, and malicious domains. %}
